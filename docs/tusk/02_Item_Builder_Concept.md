# КОНЦЕПЦИЯ: КОНСТРУКТОР ПРЕДМЕТОВ

**Статус:** Brainstorm / Concept
**Цель:** Создать систему для генерации разнообразной экипировки с уникальными свойствами и описаниями.

---

## 1. Принцип Конструктора

Вместо создания тысяч предметов вручную, мы используем систему "сборки" из базовых частей.

**Финальный Предмет = База + Материал + Модификаторы**

### 1.1 База (Base)
Определяет тип предмета и его базовые характеристики (скорость атаки, тип урона, базовый разброс).
*   **Примеры:** "Меч", "Топор", "Кожаная куртка", "Кольцо".

### 1.2 Материал (Material)
Определяет основные статы (Сила/Ловкость) и множители.
*   **Примеры:**
    *   "Железный" (Баланс).
    *   "Костяной" (Выше шанс крита, ниже прочность).
    *   "Обсидиановый" (Выше урон, но тяжелее).

### 1.3 Модификаторы (Modifiers)
Это случайные бонусы к вторичным характеристикам, которые добавляются при крафте или дропе.
*   **Механика:** Используется наша стандартная система модификаторов (`dodge`, `crit_chance`, `hp_max`).
*   **Тиры:** Чем выше редкость, тем больше слотов под модификаторы.
    *   *Пример:* `+3% Уворота`, `+50 HP`, `+2% Шанс Крита`.

---

## 2. Генерация Описания (LLM)

Нейросеть получает не сухие цифры, а **Нарративные Теги**, которые соответствуют выпавшим модификаторам.

### Логика Трансляции:
*   Модификатор `+Dodge` -> Тег `lightweight` (легкий), `balanced` (сбалансированный).
*   Модификатор `+Crit` -> Тег `razor_sharp` (острый), `serrated` (зазубренный).
*   Модификатор `+Fire_Dmg` -> Тег `warm_to_touch` (теплый), `smoldering` (тлеющий).

### Пример Генерации:
*   **Входные данные:**
    *   База: Меч
    *   Материал: Железо
    *   Модификаторы: `+Уворот`, `+Огонь`
*   **Промпт для LLM:** `[sword, iron, lightweight, smoldering]`
*   **Результат (Имя):** "Танцующий Клинок Углей"
*   **Результат (Описание):** "Удивительно легкий железный меч. Его лезвие всегда остается теплым, а при взмахе оставляет в воздухе шлейф из искр."

## 3. Типы Предметов

### Экипировка (Generated)
*   Оружие, броня, аксессуары.
*   **Всегда** генерируются по системе Конструктора.

### Расходники и Ресурсы (Static)
*   Зелья, еда, руда, дерево.
*   Имеют **статичные, заранее прописанные** характеристики и описания. Их не нужно генерировать, так как они являются строительными блоками для Конструктора.

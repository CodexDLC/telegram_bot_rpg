# Known Issues (Известные проблемы)

## Текущие ограничения

### 1. Нет кэширования Query Plan
Планы запросов перестраиваются при каждом запросе.
*   **Решение:** Можно кэшировать словарь `QUERY_PLANS`.

### 2. Нет частичных обновлений (Partial Updates)
Всегда происходит полная перезагрузка данных.
*   **Будущее:** Обновлять только измененные поля (например, только инвентарь).

### 3. Нет повторного использования временных ключей (Temp Key Reuse)
Каждый запрос создает новые временные ключи (`temp:setup:{uuid}`).
*   **Будущее:** Переиспользовать ключ, если данные не изменились и TTL не истек.

### 4. Нет фоновой очистки
Система полагается только на Redis TTL (1 час).
*   **Будущее:** Периодическая задача очистки (cleanup job).

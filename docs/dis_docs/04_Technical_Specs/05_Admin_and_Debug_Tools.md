# 05. Admin Dashboard & Utility Scripts

## 1. Admin Dashboard (Streamlit)

Вместо набора Telegram-команд, для администрирования и отладки используется полноценная веб-панель на Streamlit, расположенная в `tools/admin_dashboard`.

**Запуск:**
```bash
streamlit run tools/admin_dashboard/app.py
```

Панель предоставляет визуальный доступ к ключевым системам игры.

### 1.1. World Map (Карта Мира)
*   **Функционал:** Интерактивная карта мира, где можно выбрать любой тайл и посмотреть его свойства (ID зоны, тип местности, флаги, контент).
*   **Применение:** Отладка генерации мира, проверка проходимости, поиск аномалий.

### 1.2. Bestiary (Бестиарий)
*   **Функционал:** Просмотр всех `MonsterFamily` и `MonsterSpecies`, их базовых статов, способностей и вариантов экипировки.
*   **Применение:** Балансировка монстров, проверка корректности данных.

### 1.3. Active Clans (Активные Кланы)
*   **Функционал:** Список всех заспавненных кланов монстров на карте. Показывает состав клана, его `clan_hash`, местоположение и статус.
*   **Применение:** Мониторинг популяции монстров, отладка `ClanFactory`.

### 1.4. Analytics (Аналитика)
*   **Функционал:** Отображение ключевых метрик (TBD).
*   **Применение:** Анализ игрового баланса и активности.

### 1.5. Players (Игроки)
*   **Функционал:** Просмотр и редактирование данных игроков (инвентарь, статы, позиция).
*   **Применение:** Помощь застрявшим игрокам, тестирование.

## 2. Утилитарные Скрипты (`scripts/`)

В папке `scripts/` находятся мощные консольные инструменты для генерации, валидации и отладки.

### 2.1. `seed_world_gen.py`
*   **Назначение:** Основной скрипт для генерации игрового мира с нуля.
*   **Описание:** Создает структуру мира, ландшафт, дороги, POI и наполняет стартовые зоны контентом. Подробное описание находится в документе `02_World_Generation.md`.
*   **Режимы:**
    *   `test`: Быстрая генерация геометрии для отладки.
    *   `full`: Полная генерация с контентом.
    *   `content_only`: Только генерация контента без изменения структуры мира.

### 2.2. `validate_gamedata.py`
*   **Назначение:** Валидатор целостности игровых данных.
*   **Описание:** Этот скрипт запускается для проверки того, что все "магические строки" и ссылки в конфигах корректны.
*   **Проверки:**
    *   Все ли предметы, указанные в `loadout_options` у монстров, существуют в библиотеке предметов.
    *   Корректны ли ID биомов и террейнов.
    *   Нет ли сломанных ссылок в других игровых данных.
*   **Применение:** Обязателен к запуску перед коммитом изменений в `game_data`.

### 2.3. `validate_world.py`
*   **Назначение:** Валидатор сгенерированного мира в БД.
*   **Описание:** Проверяет логическую целостность карты после генерации.
*   **Проверки:**
    *   Существуют ли "острова" (непроходимые участки, которые должны быть проходимы).
    *   Корректно ли проложены дороги.
    *   Нет ли зон с аномальными характеристиками.
*   **Применение:** Используется для отладки `seed_world_gen.py`.
